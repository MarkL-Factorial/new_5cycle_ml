cell_lifetime weekend build — Phases 1+2+3 summary

Branch: feature/cell_lifetime  ·  Completed: 2026-05-16 in-session  ·  Source repo: MarkL-Factorial/new_5cycle_ml @ feature/cell_lifetime

A new sibling package cell_lifetime/ adds three modeling capabilities on top of cell_classifier/ (which stays untouched per the inviolable no-modify-existing rule):

  • Classification — XGBoost binary pass/bad predictor (mirror of cell_classifier's RF baseline)
  • Regression — XGBoost and EBM cycle-life regressors on faded-only cells, with Box-Cox / sqrt / log target transforms
  • Survival — XGBoost-AFT and Random Survival Forest, censoring-aware on all 415 cells

All work is gated by 47 unit + integration tests, and validated against real ml_label_preprocess/datasets/A2.2_b1 data at N=300 (1 seed).

Regression — cycle life on faded cells (n=187, untransformed cycle scale)

Model Transform Test MAE (cycles) Runtime
xgb_regressor sqrt (≈ Box-Cox λ=0.534) 129.7 7.6s
ebm_regressor boxcox (λ=0.534 on real data) 138.3 64.8s

Target-transform note: the cycle-life target on A2.2_b1 has median 317, range 6–1052. We confirmed Box-Cox's optimal λ is 0.534 — essentially sqrt (λ=0.5). log over-corrects (post-transform skew −1.12 vs sqrt's −0.26). The configs default to sqrt for xgb_regressor and boxcox for ebm_regressor; log is available but not recommended for this data.

Survival — censoring-aware on all 415 cells (187 observed + 228 censored)

Model C-index AUC@200 AUC@300 AUC@400 Runtime
xgb_aft (objective=survival:aft) 0.778 0.914 0.842 0.831 6.8s
rsf (RandomSurvivalForest) 0.807 0.911 0.880 0.896 11.8s

RSF beats XGB-AFT at every horizon in this 1-seed smoke. The nonparametric forest is a better fit than AFT's parametric (normal / logistic / extreme) assumption on this data.

Key design decisions

1. Sibling package, not a refactor

The inviolable rule "never modify existing scripts" forced cell_lifetime to live entirely under a new sibling directory. cell_classifier, ml_label_preprocess, and root files are read-only consumers; cell_lifetime imports from them via pip install -e but never edits them.

2. Workspace 10-core cap

Initial smoke with n_jobs=-1 produced 8000% CPU and 431 threads on the 200-row dataset — XGBoost's OMP threads colliding. All models now declare n_jobs=10 (XGB) / nthread=10 (xgb.Booster), and scripts export OMP_NUM_THREADS=10 OPENBLAS_NUM_THREADS=10 MKL_NUM_THREADS=10 NUMEXPR_NUM_THREADS=10 to enforce the cap at the C-library level.

3. n_regular >= 6 cohort filter

The loader drops cells with n_regular < 6 (415 cells survive on A2.2_b1). Upstream ml_label_preprocess only filters at n_regular >= 5; we tighten to 6 because cells with very few cycles haven't yet expressed their degradation behaviour.

4. risk_orientation for survival sign normalization

XGB-AFT predicts log-time (higher = later failure); RSF predicts risk (higher = sooner failure). To compute C-index cleanly across both, CycleLifeModel.risk_orientation ∈ {"time_high", "risk_high"} declares the convention, and the pipeline's _normalize_risk() negates AFT's predictions before passing to concordance_index_censored.

5. SHAP deferred for survival models

shap does not natively support xgb_aft (log-time interpretation is non-trivial) or sksurv.RandomSurvivalForest. Both models' compute_shap() returns None and the pipeline's shap stage handles that. Permutation importance still works for both — that's the recommended explanation path for survival models.
